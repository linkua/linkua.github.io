QEMU

QEMU” 是一个广泛使用的开源计算机模拟器和虚拟机。”

  • 当作为模拟器时，可以在一种架构（如 x86 PC）下运行另一种架构（如 ARM）下的操作系统和程序。通过使用动态转换，它可以获得非常好的性能。
  • 作为虚拟机时，QEMU 可以使用其他虚拟机管理程序（如 Xen 或 KVM）来使用 CPU 扩展（HVM）进行虚拟化，通过在主机 CPU 上直接执行客户机代码来获得接近于宿主机的性能。

LVGL

LVGL（Light and Versatile Graphics Library）是一个免费的开源图形库，提供创建具有易于使用的图形元素、漂亮的视觉效果和低内存占用的嵌入式 GUI。接下来我们来看一下 LVGL图形用户库的主要特征有哪些：

  • lvgl 其实就是各种图形部件 Widget（如滑块、滚轮、按钮等）的集合库（GUI 库）。每个部件都是通过 API 函数创建（函数格式一般都是 lv_xxx_create( )，xxx 是部件名）。lv_obj_t 是部件类型，是结构体进行 typedef 后的重命名（可用来创建结构体或结构体指针，这个结构体内部具备所有部件的属性），因此，所有部件都具备一些共同属性，例如大小、颜色、位置、样式这些，只是不同部件其具体的参数可能不同

  • lvgl 采用了一定的面向对象编程的设计思想，我们操控的都是一个个别人事先为我们准备好编写好的部件（对象），如果想要使用部件，就得先创建部件需要调用部件创建函数（实例化），不像 C 是面向过程的编程，不管做什么都是从零开始（想使用部件也得从零开始编写出一个部件）。所有的对象都是在 lv_obj_t 这个结构体的基础上进行演变的，诞生各种不一样的部件

  • 可扩展性：它能够以小内存运行(最低 64 kB 闪存，16 kB RAM 的 MCU)。 需要32kB RAM 和 128kB ROM, 1/10 屏幕 framebuffer size。

ESP32 简介

ESP32芯片具有以下特性：

  • 处理器和内存：双核32位LX6微处理器，主频最高可达240MHz，520KB SRAM，448KB ROM，同时支持外部4MB SPI Flash和外部8MB PSRAM，为数据的处理和存储提供强大的支持。
  • WiFi：支持802.11 b/g/n（2.4GHz）标准，支持STA/AP/STA+AP工作模式，最大传输速率可达150Mbps，并支持WPA/WPA2/WPA3安全模式，提供稳定、安全的无线网络连接。
  • 蓝牙：支持蓝牙4.2（包括BR/EDR和BLE），支持蓝牙音频传输（A2DP/AVRCP/SPP），使得无线音频传输和远程控制更加便捷。
  • 外设和接口：ESP32提供了丰富的外设和接口，包括最多34个可编程GPIO引脚，支持UART、SPI、I2C、I2S、PWM等多种通信协议。此外，还集成了12位ADC、8位DAC、触摸传感器、霍尔传感器等，为各种应用场景提供了强大的支持。
  • 电源管理：具有多种低功耗工作模式，电源电压范围2.2V-3.6V，内置低压稳压器（LDO）和开关电源管理单元（SMU），实现高效的电源管理

ESP8266

ESP8266是一款低成本、低功耗、高度集成的Wi-Fi芯片，其主要参数包括：12

主频：80 MHz (单核)
内存：80 KB RAM + 4 MB Flash
Wi-Fi标准：802.11 b/g/n 不支持蓝牙
工作电压：3.0V - 3.6V
工作温度：-40℃ ~ 125℃
支持模式：STA/AP/AP+STA
支持协议：TCP/UDP
支持连接：多连接
支持升级：OTA升级
此外，ESP8266还具备以下特性：

无线参数：支持802.11 b/g/n标准，频率范围2.4GHz至2.5GHz，具有不同的发射功率和接收灵敏度，以满足不同标准下的通信需求。
存储系统：配备有32Mb(4M)的Flash存储器和80KB的SRAM，Flash存储器分为几个部分，包括Bootloader等。
处理器和性能：内置超低功耗的Tensilica L106 32位RISC处理器，最高时钟速度可达160MHz，支持RTOS和Wi-Fi协议栈，能够将高达80%的处理能力留给应用编程和开发。

ESP8266模块有多个型号和变种，例如ESP-01、ESP-12E、NodeMCU等，每种型号的功能和特点可能略有不同。

ESP32-C3发布已经有一段时间了，作为ESP8266的替代品，上架之初C3模组就保持了10元以下的友好价格，并在8266的基础上增加了蓝牙支持。
内建的USB-CDC支持，提供一个USB串口以及JTAG调试器。这表明我们可以跳过CH340、CP2102等USB转串口芯片直接对C3模组进行开发调试。

MCU 与 MPU

Posted on 2024-04-12 Edited on 2024-05-20 In MCU MPU

MCU 与 MPU

MCU的全称是Mirco Controller Unit，MPU的全称叫Micro Processor Unit。
微控制器（单片机，Cortex-M0/M3内核架构，没有内存管理单元MMU），早期8位8051系列主频不过10几MHz，flash 1-10k, ram 256/512字节， 还是12个周期执行一条指令。而随着ARM的32位MCU的出现，采用ARM的M系列的MCU也开始逐步扩大市场。以ST、NXP公司的产品为主要代表，stm32fl103, flash 32-128k, ram 10-48k。但是这些ARM系列MCU的主频一般也是在几十MHz到100多MHz的量级。
随着物理网发展，一些带有wifi/蓝牙的SOC,弥补MCU与MPU的空白，例如 ESP3 频率在240MHz, 520KB SRAM，448KB ROM，同时支持外部4MB-16 SPI Flash. 还有树莓派 pico rp2040,双核 Arm Cortex-M0 + @ 133MHz
芯片内置 264KB SRAM 和 2MB 的板载闪存。。

而微处理器（嵌入式，Cotex-A内核架构，主要指应用型处理器），其实就是微型化/集成化了的处理器。准确的说是微型化/集成化的“中央处理器”。MPU从一开始就定位了具有相当的处理和运算能力，一帮需要运行较大型的操作系统(如linux、opentwrt)来实现复杂的任务处理。因此这就决定了MPU应该具备比较高的主频，和较为强大的运算能力和内存管理单元MMU。MPU很早就演进到了32位处理器，现在更是开始大力普及64位。同时MPU也一直追求实现较高的主频。早期经典的Arm 9系列MPU频率就在200MHz-400MHz。现在手机上使用的高端MPU更是到达了3GHz，和主流的桌面处理器是一个级别了。和通用的桌面处理器一样，MPU现在也普遍“多核化”。

FREERTOS 常用于 MCU, LINUX常用于MPU, LINUX需要硬件支持MMU, LINUX的内核空间与应用程序空间分开,进程独享全部内存空间，而FREERTOS与任务同一空间（单进程），只做线程任务调度。例如路由器使用的就是MPU，通常运行openwrt,要求4MB flash和32MB RAM.

linux 发行版本

Debian GNU/Linux

Ubuntu 基于 Debian，并且相比 Debian 而言更加新手友好。而 Debian 的开发周期更慢，它的 Stable 分支也更加稳定。
不同的发行版、不同的分支的软件源不能混用。向 Debian 添加 Ubuntu 或 Ubuntu PPAs 的源可能会导致软件依赖的混乱。

CentOS/Fedora

「红帽系」的 Linux 发行版与 Ubuntu 等 Debian 系列的发行版最直观的区别就在于它们使用的软件包管理方式不一样。一般使用 dnf（推荐）或者 yum 来进行软件包管理。dnf 和 yum 都是使用 Python 编写的程序，谨慎变动系统 Python 环境，以免出现问题。
SELinux 是由 NSA 编写的开源的 Linux 安全模块，在 CentOS 和 Fedora 上都默认开启

内存需求

以下是一些常见Linux发行版的最低内存需求：

  • Ubuntu - 尽管没有明确的最低内存需求，一般来说，运行Ubuntu桌面环境至少需要512MB内存。

  • Debian - 同样没有明确的最低需求，但运行基本系统通常需要128MB内存。

  • CentOS - 官方文档建议至少有256MB内存用于运行基本系统，尽管它主要用于服务器应用，可能需要更多内存。

  • Fedora - 推荐至少有256MB内存用于基本系统，随着桌面环境的加入，可能需要更多。

  • Arch Linux - 因为它是为高度定制而设计的，所以最低内存需求取决于你的具体配置和需求。
